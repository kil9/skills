현재 저장소를 분석하여 README.md, AGENTS.md 를 신규 작성하고 CLAUDE.md 심링크를 만든다.
$ARGUMENTS 가 있으면 프로젝트 설명 힌트로 사용한다.

**작업 계획(태스크)은 이 스킬이 만들지 않는다** — `/init-backlog` 몫이다(§작업 계획 참조).

파일이 이미 존재하면 덮어쓰기 전에 사용자에게 확인한다.

## 사전 조사

저장소 구조·의존성·엔트리포인트·git 이력을 조사한다.

## 사용자 인터뷰

**문서를 작성하기 전에 반드시 수행한다.** 사전 조사로 파악되지 않은 모든 디테일을 사용자 인터뷰로 확정한다.

### 인터뷰 규칙

- 질문은 `AskUserQuestion` 선택지 형태로 제시한다(줄글 자유 입력 금지). 사전 조사 결과로 가장 가능성 높은 후보를 선택지로 만들고, 관련 질문은 한 호출에 묶는다.
- 궁금증이 전혀 남지 않을 때까지 몇 번이고 반복한다. 애매한 항목을 추측으로 채우지 않고, 답변으로 생긴 후속 의문도 다시 묻는다.
- 충분해지면 확정 항목을 요약해 사용자에게 최종 확인을 받은 뒤 문서 작성에 들어간다.

### 물어볼 만한 예시 항목

각 항목은 `AskUserQuestion`의 선택지로 변환해서 묻는다.

- 프로젝트가 해결하려는 문제와 타깃 사용자
- 실행 환경(OS, 런타임 버전, 컨테이너 여부 등)과 배포 방식
- 빌드·실행·테스트·린트 명령어
- 외부 의존 시스템(DB, API, MCP 서버 등)과 필수 환경변수
- 보안·컴플라이언스 주의사항
- 현재 개발 단계와 다음 마일스톤(문서엔 넣지 않고, §작업 계획 에서 `/init-backlog` 안내에 쓴다)

## README.md

프로젝트 설명과 기본 사용법만 담는다. 기술 중립적으로 작성한다.

- 한 줄 설명
- 설치 및 실행 방법
- 주요 기능 목록

## AGENTS.md

AI 에이전트가 이 저장소에서 작업하는 데 필요한 모든 정보를 담는다.

- 이 저장소의 프로젝트 구조, 빌드/실행/테스트 명령어, 환경변수, 보안 주의사항을 추가한다.
- 커밋 규칙: 커밋 전 README.md·AGENTS.md 를 검토하고, 변경이 필요하면 같은 커밋에 포함한다.
  backlog 를 쓰는 저장소면 태스크 파일의 상태·notes 변경도 같은 커밋에 담는다.

## 작업 계획 (backlog 위임)

**여기서 PLAN.md 를 만들지 않는다.** 태스크 진행의 진실원본은 `backlog/` 이며, 신규 저장소에
레거시 PLAN.md 를 만들면 나중에 `/migrate-to-backlog` 로 되돌리는 왕복이 생긴다.

- 위 인터뷰에서 파악한 개발 단계·다음 마일스톤을 사용자에게 요약해 주고, 계획 수립은
  **`/init-backlog` 로 이어가도록 안내한다**(이 스킬이 대신 호출하지 않는다).
- 사용자가 backlog 를 쓰지 않겠다고 **명시적으로 요청할 때만** 레거시 PLAN.md 경로를 쓴다.
  그 경우도 이 스킬이 직접 쓰지 않고 `/make-a-plan` 에 위임한다.

## CLAUDE.md (심볼릭 링크)

스킬 마지막 단계에서 수행한다. `CLAUDE.md` 를 `AGENTS.md` 로 향하는 **심볼릭 링크**로 생성한다 (일반 텍스트 파일이 아니다).

- `CLAUDE.md` 가 이미 존재하면 (일반 파일이든 심볼릭 링크든) 건너뛴다.
- 존재하지 않으면 아래 명령으로 생성한다.

```bash
ln -s AGENTS.md CLAUDE.md
```
