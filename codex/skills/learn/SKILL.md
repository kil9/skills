# Learn — 이번 세션의 학습을 영속화

**독자는 미래의 Codex 세션이다. 사용자가 아니다.**

목적은 하나다: 다음 세션이 **같은 실수를 반복하지 않게**, **같은 탐색 비용을 다시 지불하지 않게** 하는 것. 그래서 결과물은 사람이 읽는 글이 아니라 에이전트가 읽는 지침(스킬·머신 로컬 지침·AGENTS.md)이다.

> **`$publish-til` 과 혼동 금지.** 그쪽은 *사용자가* 읽고 배우는 공개 TIL 페이지를 만든다(kil9/til → GitHub Pages). 이 스킬은 *에이전트가* 배우는 쪽이며 아무것도 게시하지 않는다. 두 이름은 안 겹치지만 "배운 것"이라는 말이 같아 헷갈릴 수 있다 — 기준은 **"이 글의 독자가 누구인가"** 다. 사용자가 읽을 산문을 쓰고 있다면 스킬을 잘못 고른 것이다.

## 무엇을 남기는가

**재탐색 비용이 비싼 것만** 고른다: 탐색을 여러 번 하거나 **실패를 겪어야** 알게 된 것 — 함정, 굳힌 명령, 순서 제약, 크로스 파일 결합, 미설정 시 동작, 겉보기와 다른 실제 원인. 한두 번 탐색으로 파생되는 것은 남기지 않는다(zip-it 의 keep 기준과 동일).

남기지 않는 것: 코드를 읽으면 바로 아는 구조, git 히스토리에 있는 과거 수정, 이번 대화에서만 의미 있는 맥락, 일반적인 코딩 상식.

## 절차

1. 대화를 훑어 후보를 추린다. 없으면 그렇게 보고하고 끝낸다 — 없는데 억지로 만들지 않는다.
2. 각 후보의 목적지를 정한다:
   - **반복 절차·워크플로** → 새 스킬(`$skill-creator`) 또는 기존 스킬 SKILL.md 보강.
   - **이 머신에서만 의미 있는 사실·사용자 선호·진행 상태** → `~/.codex/AGENTS.local.md`
     (머신 로컬이라 다른 머신으로 전파되지 않는다).
   - **이 repo 의 비자명한 운영 지식** → repo AGENTS.md.
   - **머신 무관 범용 규칙, Codex 하네스 자체의 동작·함정** → 글로벌 `~/.codex/AGENTS.md`
     (실체는 `~/kil9conf/codex/AGENTS.global.md`, 심링크). **매 세션 컨텍스트에 통째로 실리므로
     짧게 쓴다** — 길어질 내용이면 repo AGENTS.md 나 `AGENTS.local.md` 가 맞다.
3. 후보가 여럿이거나 목적지가 애매하면 `request_user_input` 으로 무엇을 어디에 남길지 확정한다.
4. 확정된 것만 기록한다. 문서·스킬 편집은 zip-it 기준(정보 단가)을 지킨다.

## 쓸 때

- **미래의 나에게 지시하듯** 쓴다. "~였다"(회고)가 아니라 "~하라 / ~를 먼저 확인하라"(지침).
- 실패를 겪어야 아는 것이면 **증상 → 원인 → 대응**을 한 줄로 압축한다. 증상을 빼면 미래 세션이 그 지식을 꺼내 쓸 트리거를 잃는다.
- 이미 같은 내용을 담은 파일이 있으면 새로 만들지 말고 그 파일을 고친다.
- **검증하지 않은 가설을 규칙으로 굳히지 마라.** 근거(명령 출력·이슈 번호·파일 라인)를 함께 남기고,
  확인하지 못했으면 "가설"이라고 표시한다. 여기서 고치는 파일은 세션을 넘어 살기 때문에, 틀린 한 줄이
  미래의 모든 세션을 오염시킨다. 그럴듯한 추론일수록 위험하다 — 다음 세션은 그걸 기정사실로 읽는다.
