마일스톤 **전체 진척을 한 화면에** 보여주고 다음에 손댈 마일스톤 하나를 추천한다. `$next-task` 의 마일스톤 판(版)이다 — 저쪽이 "다음에 할 태스크"를 고른다면 이쪽은 "지금 어디까지 왔고 어느 덩어리를 밀지"를 본다. **읽기 전용**이다: 태스크·플랜·코드를 건드리지 않고 보고만 한다. 근거는 backlog 데이터(레거시는 플랜 본문)뿐 — 코드·이슈·git·웹은 조사하지 않는다.

개별 태스크 단위 추천은 하지 않는다. 마일스톤을 고른 뒤 그 안에서 무엇을 할지는 `$next-task` 몫이라 역할이 겹치지 않게 한다.

## 0. 모드 감지

repo 루트에 `backlog/` 디렉터리(또는 `backlog/config.yml`)가 있으면 **backlog 모드**(§1-3), 없으면 **레거시 모드**(§4)로 동작한다. backlog 모드가 기본이다.

**CLI 없으면 중단.** backlog 모드면 `command -v backlog` 로 확인한다. 없으면 태스크 파일을 직접 읽어 폴백하지 말고(상태·의존·마일스톤을 잘못 읽는다) `bash "${K9HOME:-$HOME/kil9conf}/bootstrap/install-backlog-md.sh"` 설치를 안내한 뒤 멈춘다.

## 1. 마일스톤 수집 (backlog 모드)

```
backlog milestone list --plain --show-completed
```

`--plain` 은 필수다(보드/브라우저 실행 금지). `--show-completed` 가 없으면 완료 마일스톤이 `(collapsed, use --show-completed to list)` 로 접혀 대시보드에 안 나온다 — 전체 진척이 목적이므로 항상 붙인다.

출력은 두 그룹이다.

```
Active milestones (5):
  m-1: skills repo 분리·공개 (0/1 done)
  m-4: F13 STT 재부팅 내성 (5/6 done)

Completed milestones (2):
  m-6: vial 치트시트 인쇄 조판 개선 (7/7 done)
```

각 줄에서 **id·제목·완료/전체 카운트**를 얻는다. 이 한 번의 호출로 진척 바에 필요한 건 다 나온다 — 카운트를 얻으려고 태스크를 세지 말 것.

마일스톤이 하나도 없으면 그 사실을 알리고 `$add-milestone` 을 안내한 뒤 멈춘다.

## 2. 착수 가능 / 막힘 분해 (추천 근거)

진척 바만으로는 **막힌 마일스톤과 그냥 안 한 마일스톤이 구별되지 않는다**. 5/6 이 남은 하나가 사용자 입회 대기라면 추천해봐야 소용없다. 그래서 미완료 마일스톤에 한해 태스크를 분해한다.

```
backlog task list --plain
```

여기서 Done 태스크 ID 집합과 미완료(To Do / In Progress / Blocked) 태스크 목록을 얻는다. 그다음 **미완료 태스크만** 개별 조회한다(Done 은 조회하지 않는다 — 카운트는 §1 이 이미 줬다).

```
backlog task <id> --plain
```

`Milestone:` 로 어느 마일스톤 소속인지, `Dependencies:` 로 의존을, `Status:` 로 상태를 읽는다. 판정 규칙은 `$next-task` §3 과 **같다**:

- **In Progress** → 착수 가능(이어서 할 일).
- **To Do** 이고 의존이 비었거나 나열된 ID 가 모두 Done 집합에 속함 → 착수 가능.
- **To Do** 인데 의존이 안 풀림 → 막힘.
- **Blocked** → 막힘. 사유는 `Implementation Notes:` 첫 줄.

마일스톤별로 `착수 가능 N개 / 막힘 M개` 를 집계한다. 마일스톤에 속하지 않은 태스크는 집계에서 빼되, 개수가 있으면 대시보드 아래 한 줄로 알린다(`마일스톤 없음: 3개`).

## 3. 대시보드와 추천

완료된 마일스톤까지 포함해 **전부** 한 화면에 나열한다. 개괄 파악이 목적이므로 마일스톤당 한 줄을 넘기지 않는다.

```
## 전체 진척

[##########] m-6  vial 치트시트 인쇄 조판 개선   (7/7)  완료
[########--] m-4  F13 STT 재부팅 내성            (5/6)  막힘 1
[----------] m-7  herdr 안에서 키맵 띄우기        (0/1)  착수 가능 1
[----------] m-1  skills repo 분리·공개          (0/1)  막힘 1
[----------] m-2  vial 치트시트 가독성 개선       (0/0)  태스크 없음

← 추천: m-7 — 착수 가능한 태스크가 1개 있고 의존이 모두 풀렸다.
```

- 진척 바는 10칸 고정. 완료율을 반올림해 채운다. 전체가 0 이면(`0/0`) 빈 바로 두고 `태스크 없음` 을 붙인다 — **0 으로 나누지 말 것**. 마일스톤은 태스크 없이 먼저 만들어지는 일이 흔해서 이 경우가 실제로 자주 나온다.
- **제목 열을 문자 수로 정렬하지 말 것.** 한글 제목은 한 글자가 터미널 2셀이라 `%-32s` 류로 맞추면 제목 길이에 따라 뒤 열이 들쭉날쭉해진다. 셀 폭으로 세거나(East Asian Wide/Fullwidth = 2), 아예 고정폭 정렬을 포기하고 `제목 (n/m) 꼬리표` 를 공백 하나로 이어 붙인다. 후자가 안전하다 — 대시보드의 값은 정렬이 아니라 한눈에 들어오는 것이다.
- 정렬은 진척 높은 순이되 완료 마일스톤을 위로 몰지 말 것(전체를 한눈에 보는 게 목적이다). 완료분은 `완료` 꼬리표로 구분되므로 순서로 또 가를 필요가 없다.
- 꼬리표는 `완료` / `착수 가능 N` / `막힘 N` / `태스크 없음` 중 해당하는 것. 착수 가능과 막힘이 둘 다 있으면 둘 다 적는다.

**추천 하나**를 맨 아래 붙인다. 고르는 순서:

1. 진행 중(In Progress 태스크가 있는) 마일스톤 — 이미 손댄 것을 먼저 끝낸다.
2. 착수 가능 태스크가 있는 마일스톤 중 진척이 가장 높은 것 — 끝내기 가장 가까운 덩어리.
3. 그것도 없으면 추천하지 않고 "전부 막혔거나 완료" 라고 알린다. 이때 막힘 사유를 마일스톤별로 한 줄씩 보여 준다(무엇을 풀어야 진행되는지가 그 상황에서 유일하게 쓸모 있는 정보다).

추천에는 **한 줄 근거**를 붙인다. 진척 숫자만 다시 말하지 말고 왜 지금 그것인지를 적는다.

## 4. 레거시 모드 (PLAN.md)

`backlog/` 가 없으면 플랜 파일에서 `M-N` 표기를 파싱해 같은 대시보드를 낸다. 플랜 파일 위치·전문은 `next-task` 스킬 디렉터리의 `plan-context.sh --dump` 로 받는다.

```
bash ~/.codex/skills/next-task/plan-context.sh --dump
```

exit 1(플랜 파일 없음)이면 알리고 `$init-backlog` 을 안내한 뒤 멈춘다.

- `M-N` 아래 딸린 `T-N` 들의 `[x]`/`[ ]`/`[→]`/`[!]` 로 완료·착수가능·막힘을 센다.
- **`M-N` 구분이 아예 없거나 표기가 제각각이면 억지로 만들지 말 것.** 파일이 마일스톤 개념 없이 태스크만 나열하는 경우가 흔하다. 그때는 "이 플랜에는 마일스톤 구분이 없다" 고 알리고 `$next-task` 를 안내한 뒤 멈춘다. 태스크를 임의로 묶어 가짜 마일스톤을 만들면 없는 구조를 있는 것처럼 보고하게 된다.
- 플랜 파일이 여럿이면 파일별로 묶어 밝힌다.

## 5. 경계

- 읽기 전용이다. `backlog task edit`·`milestone edit` 류를 부르지 않는다. 상태가 틀려 보여도 고치지 말고 보고만 한다.
- 개별 태스크를 추천하지 않는다. 사용자가 태스크 단위를 원하면 `$next-task` 로 넘긴다.
- 여기서 멈춘다 — 마일스톤을 골랐다고 이어서 구현하지 않는다. 그건 `$start-task`·`$loop-task` 몫이다.
