$ARGUMENTS 가 있으면 커밋 메시지 힌트로 사용한다.

## 0. 컨텍스트 수집 (셸 명령 1회)

이 SKILL.md 와 같은 디렉터리의 `commit-context.sh` 를 실행해 status·최근 커밋 스타일·diff·
untracked 내용을 한 번에 받는다. status/diff/log 를 따로따로 조회하지 않는다 — 절단된 diff 등
부족한 부분만 추가 조회한다.

## 1. 문서 갱신 판정

커밋 전에 두 파일 모두 판정을 마친다:

- 프로젝트 구조·실행 명령·아키텍처가 바뀌었으면 → AGENTS.md 수정 후 커밋에 포함
- 사용자 대상 실행 방법·구성 설명이 바뀌었으면 → README.md 수정 후 커밋에 포함
- 단순 버그 수정·리팩터링·결과 파일 갱신이면 → 수정하지 않음

## 2. 스테이징 및 커밋 (셸 명령 1회)

스테이징과 커밋은 `git add <파일들> && git commit ...` 한 번의 명령으로 묶는다.

- 파일을 명시적으로 지정해 스테이징한다 — `git add -A` 금지. 민감 파일(`.env`, 인증정보)은 제외.
- 커밋 메시지는 한국어, 아래 형식:

  ```
  변경 내용 한 줄 요약 (72자 이내)

  - 변경 항목 1
  - 변경 항목 2
  ```

  추가 설명이 필요하면 목록 아래 빈 줄 후 서술한다.
