# Zip It

에이전트가 읽는 문서의 정보 단가를 높인다. 짧게 만드는 것 자체가 목표가 아니며, 강도는 문서마다 적응적으로 정한다: 전부 삭제해도 되고, 압축이 전혀 안 돼도 되고, 빠져 있던 값진 지식이 있으면 오히려 길어져도 된다.

## 판단 기준

실행 시점의 에이전트는 그때의 신선한 상황을 직접 탐색해 스스로 판단할 수 있다. 나중에도 쉽게 판단할 수 있는 내용을 미리 적어둘 필요는 없고, 탐색을 여러 번 하거나 실패를 몇 번 겪어야 알게 되는 지식은 적어두는 쪽이 싸다.

- **삭제**: 탐색 한두 번으로 파생되는 것. 디렉터리 트리와 파일별 한 줄 설명, Makefile 타깃이나 CLI --help 나 스크립트 usage 를 옮겨 적은 표, 워크플로 파일 목록, 모델이 이미 아는 일반 지식(표준 도구 사용법, 튜토리얼조 설명). 진실의 원본이 따로 있으면 한 줄 포인터로 치환한다.
- **유지, 필요하면 보강**: 실패 유래 지식. 외부 엔드포인트와 인증 방식의 함정, 디버깅 끝에 얻은 정확한 명령, 순서 제약, 한쪽을 바꾸면 다른 쪽도 고쳐야 하는 크로스 파일 결합, 조용히 실패하는 함정, 미설정 시 동작 같은 의미론. 문서에 빠져 있지만 대화나 이력에서 확인되는 이런 지식은 새로 적는다.
- **무손실**: 행동 규칙(기본 동작 override), 승인 게이트, 안전/보안 규칙, 스킬 frontmatter 의 description(트리거 매칭에 쓰인다). 문구는 다듬어도 규칙의 의미는 잃지 않는다.
- **검증 후 정정**: 낡아 보이는 사실 서술은 현재 상태와 대조한다. 싸게 검증되면 고치고, 검증이 비싸면 주장을 빼거나 발견 사항으로 보고만 한다.

## 절차

1. 대상 결정: 인자 경로가 있으면 그것. 없으면 cwd 의 AGENTS.md (CLAUDE.md 가 심링크면 원본 쪽을 수정). 스킬이 대상이면 SKILL.md 와 딸린 참조 파일까지 포함한다.
2. 대상 전체와, 대상이 중복 서술하는 진실의 원본들을 읽는다.
3. 청크 단위로 위 기준에 따라 분류하고 재작성한다.
   - 삭제 대상 위치에 있지만 실패 유래인 지식은 버리지 말고 제자리로 이전한다(코드 주석, .env.example, 운영 문서, 에이전트 메모리 등).
   - 같은 종류의 비대화가 재발할 문서면 "여기에 X 를 유지하지 않는다" 한 줄로 방지선을 친다.
4. 스킬이 대상이면 추가로: 모델이 이미 할 줄 아는 방법 설명은 지우고, 이 환경에서만 통하는 정확한 절차와 제약만 남긴다. 예시는 하나면 충분하다. 다른 도구용 미러나 사본이 있으면 압축된 원본 기준으로 함께 갱신한다.
5. 독립 동작해야 하는 문서(글로벌 규칙 없이도 동작해야 하는 AGENTS.md 등)는 행동 규칙 코어를 유지한다.
6. 검증: 남긴 사실 서술이 현재 상태와 맞는지, 무손실 대상이 빠지지 않았는지 전후 내용을 대조해 확인한다.
7. 보고: 전/후 바이트, 삭제/유지/보강/정정 요약, 범위 밖 발견. 커밋은 사용자가 요청할 때만, 대상 repo 의 커밋 규칙대로 한다.
